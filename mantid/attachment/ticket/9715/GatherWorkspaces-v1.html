160<h2><a class="toc-backref" href="#id3">Description</a><a class="headerlink" href="#description" title="Permalink to this headline">Â¶</a></h2>
161<p>GatherWorkspaces is only availble for MPI builds.
162It stitches together the input workspaces provided by each of the processes into a single workspace in the root process.
163The spectra in the output workspace will be ordered by the rank of the input processes.
164It is up to the caller to ensure this results in the required ordering.
165Furthermore, there are all sorts of things that ought to be consistent for this algorithm to make sense (e.g. the instrument). The general philosophy, though, is to leave the responsibility for this to the user and only check the vital things (i.e. that the number of bins is consistent).</p>
