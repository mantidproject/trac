Description

Some issues have been fixed with the connected component analysis, and some others have been discovered as part of #9509. Generally the CCL works well, and only a very small number of pixels are misplaced.

The issues seem to stem from resolving a unified label id across neighbouring pixels (this is not a multi-threaded issue). This is probably due to the fact that the neighbours are being stored as independent copies, so updates to one parent DisjointElement, don't affect other connected elements (when they should). I therefore suggest that the solution to this is to:

a) Prove my theory above. A few unit tests in CCL should do this

b) If (a) is proved. Fix it by storing smart_pointers in the neighbours array.
