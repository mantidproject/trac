MUON: Disable "Load Current Run" Button When Loading Files

Reported by: Peter Parker Owned by: Peter Parker
Priority: critical Milestone: Release 2.2
Component: Mantid Keywords:
Cc: anders.markvardsen@… Blocked By:
Blocking: Tester: Anders Markvardsen

Description

Currently the "previous" and "next" arrow buttons of the Home tab of the Muon Analysis interface are disabled when they are pressed, for the duration of loading a run.

There is a problem where pressing "Load current run" and then the previous button immediately afterwards (or vice-versa) sometimes causes Mantid to load multiple runs one after the other in a continuous loop.

A good solution would be to disable "Load current run" (and possibly even the MWRunFiles widget) when loading runs with the interface.
