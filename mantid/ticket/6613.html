Priority: blocker Milestone: Release 2.5
Component: Mantid Keywords:
Cc: stephen.cottrell@… Blocked By:

Description (last modified by Anders Markvardsen) (diff)

At some point recently MuonAnalysis interface has stopped restoring allow plotting again.

As reported by Steve: Inserting a nonexistent detector into the grouping correctly generates an error box and disables plotting, but plotting is not restored when the grouping is corrected

My testing of this: Could reproduce the above, although I found that in some cases it did restore

More worryingly I found the following, I think related bug, which has somehow crept in:

Open muon interface

click in the Load run text box

click somewhere else - this now disable the interface entirely...!

To test do:

1 Open muon interface

2 click in the Load run text box

3 click somewhere else - this should not disable the interface entirely

and

4 Load any muon data, e.g. any of the MUSR data from AutoTestData

  1. go to Grouping Options tab
  1. change the Detector IDs (2nd column) for one of the group to anything invalid and revert this change back to the valid string. What you should observe is that on the home tab the Load Run text box can still be edited (i.e. is not greyed out)
