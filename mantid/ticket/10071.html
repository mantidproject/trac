Separate monitors into a subworkspace when loading live data.

Reported by: Roman Tolchenov Owned by:
Priority: major Milestone: Backlog
Component: Framework Keywords:
Cc: Blocked By:
Blocking: Tester:

Description

For some instruments at ISIS the monitors can have a time regime (binning, number of bins) different form those of the data spectra. In this case the monitors cannot be put into the same workspace with the data.

When loading live histogram data the monitors should be stored in a subworkspace (monitor workspace) of the data matrix workspace. If we are loading a list of spectra mixing of time regimes isn't allowed, throw an error.
