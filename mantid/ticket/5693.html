MantidPlot: ColorMap does not revert to default automatically.

Reported by: Michael Reuter Owned by: Jay Rainey
Priority: critical Milestone: Release 2.6
Component: MantidPlot Keywords: Student,First
Cc: Blocked By:
Blocking: Tester: Michael Reuter

Description

If one uses the custom colormap to switch and then try to switch back to the default colormap, MantidPlot must be restarted in order for this to take effect. I don't know of many programs that require a restart to change a colormap. I've observed this behavior on both Windoze (install kit) and Linux (dev version).

OK, the color map switching seems to work. However, it looks like a bug in the custom colormap dialog was introduced. To see the issue, do the following with the layer properties dialog open:

  1. Click on Custom Color Map and then the Select ColorMap button
  2. Select a colormap in the Pick a Colormap dialog and click Open
  3. Dialog opens again, so I have to select and click again.
  4. Dialog closes, hit Apply, see colormap change.
  5. Click on Default ColorMap, hit Apply and colormap changes.
  6. Click on Custom Color Map and then the Select ColorMap button
  7. Select a colormap in the Pick a Colormap dialog and click Open
  8. Dialog opens again, and I have to click Open 3 more times to get it to select colormap
  9. Dialog finally closes, hit Apply, see colormap change.

If I repeat steps 5-7 and then try to repeat step 8 it requires 6 total clicks on Open to get the Dialog to finally close. Each iteration increases the number of clicks required by 2. I see this behavior on Ubuntu and RHEL6.
