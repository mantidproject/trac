Description

From Steven:

  1. The 'Results' tab and table seem to work fine for the first sequential fit, but appear to break in various ways on subsequent fitting. I'm unable to give you an exhaustive description of the problem, but here are a couple examples:

a) Fit EMU 45505 individually with an exponential; fit sequence EMU 45505-45508; select sequential fit runs in 'results' and 'field_danfysik'; table 'resultstable1' generated ok. Sequential fit EMU 45505-45509 under a new label; 14 runs in results table; generate results table including only the 5 runs under new label ... table only has one run.

b) Fit EMU 45505 individually with an exponential; fit sequence EMU 45505-45509; select sequential fit runs in 'results' and 'field_danfysik'; table 'resultstable1' generated ok. Reload EMU 45505, change model (two exponentials) and fit individually; sequential fit EMU 45505-45509 under a new label; 16 runs listed in results tab - label1_run *2 + label2_run * 1 (i.e. an unexpectedly large number of runs are listed). Tick to include only label2 runs in the results table as well as 'field_danfysik' ... error generated 'Please select options from both tables' and no table is generated.
