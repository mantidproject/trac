Description

To reproduce the error(s):

  1. Download and install paraview in the default location
  2. Download and install mantid in the default location
  3. Download and install mantid_vates in the default location

During the vates installation it pops up a dialog 'Mantid does not appear to be installed'. Press 'OK' to clear the dialog and the installer will say that it succeeded. Now:

  1. Start mantid
  2. OneStepMDEW("TOPAZ_3131_event.nxs", "TOPAZ_3131")
  3. Right click on the MD workspace to try to bring up the vates simple gui
  4. See the new error dialog "Sorry, MantidPlot has caught an unexpected exception: InterfaceManagerImpl::createVatesSimpleGui..."

Otherwise you can start the vates simple gui directly (with the failed vates install still) which brings up an error dialog "...PV_PLUGIN_PATH not setup....".

What should happen is a simple:

  1. Install mantid
  2. Install paraview
  3. Install mantid_vates
  4. Everything works

The extra environment variable hacking will stump users.
