comment:44 Changed 7 years ago by Nick Draper

  • Description modified (diff)

Tester:

Do not worry about the length of the comments in this ticket, it really isn't that complicated.

Only 3 files affected, A new PythonAlgorithm, a test for it, and a warning fix on MantidDock. To test the PythonAlgorithm LoadLogPropertyTable, use the musr files in autotestdata, or any other range of files.

It should:

  1. Create a tableworkspace of log values against the run number.
  2. Load any file type that Load can handle.
  3. Not crash with multiperiod data - although values will be from period 1
  4. Handle gaps in the file structure (although this can be slow over a network if you choose a range of 100s), there are emu files in autotest data that are used in the unit test for this.
  5. Load only a single spectra of the data (if the file loader supports this).
  6. Print out the list of acceptable log names if one is entered incorrectly.
  7. Use a hidden workspace for the temporary loaded workspaces, and clean up after itself.
  8. Handle multiple logvalues in a single call, the log values can be entered as a comma separated list.
