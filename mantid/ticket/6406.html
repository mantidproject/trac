comment:24 Changed 6 years ago by Federico M Pouzols

  • Status changed from verifying to closed

Test environment: Windows 7 64bit
Windows installer 3.2.20141208.1833

Coud not find any serious issue.

I just found a bit confusing that the "send to" menu is not shown by default. This happens because the only program/item listed in the "send to" options is SansView, which uses SaveCanSAS1D which in turn requires a particular type of workspace (with units of MomentumTransfer). If you add other programs using more generic Save algorithms, such as SaveNexus or SaveAscii, everything seems to work well. This together with the fact that "send to" is only visible in Settings -> Mantid -> Send to, may hide the "send to" functionality from many users. An alternative would be to show the incomplete/wrongly configured "send to" programs in the menu but as disabled entries (as it is done in the clear options menu). Added this as a minor ticket: #10729

A few related issues in MantidPlot are listed in ​http://trac.mantidproject.org/mantid/ticket/6405#comment:26, ​http://trac.mantidproject.org/mantid/ticket/6404#comment:27, ​http://trac.mantidproject.org/mantid/ticket/6400#comment:26.
