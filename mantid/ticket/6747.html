Last modified 5 years ago

Improve Monte Carlo simulation in LeBailFit and enable to refine background

Reported by: Wenduo Zhou Owned by: Wenduo Zhou
Priority: major Milestone: Release 2.5
Component: Mantid Keywords:
Cc: petersonpf@… Blocked By:
Blocking: Tester: Vickie Lynch

Description (last modified by Wenduo Zhou) (diff)

Improve the Monte Carlo simulation algorithm in LeBailFit. Here is a list of work in mind.

  1. As a new value of a parameter proposed, it will be checked against the specified boundary of this parameter. There can be 3 ways to treat the situation that the new value is out of boundary, (a) half distance to the boundary, (b) reflect from boundary, and (c) periodic boundary. A random number can be used to determine which method to use;
  1. Enhance the book keeping of parameter values in Monte Carlo algorithm.
  1. The background is refined outside of LeBailFit presently. Enable LeBailFit to refine background polynomail parameters. It might improve the refinement.

How to verify this ticket

It can be divided into 2 tests to verify this ticket.

Refine background

  1. Download the following files from this ticketâ€™s attachment.
    • NOM_11848-4.dat
    • LB4853b2.hkl
    • nom_bank4.irf
    • setup_preview-1.py
    • refine_background.py
  1. Launch MantidPlot
  1. Run setup_preview.py to set up the inputs of LeBailFit()
    • You will notice that Rwp = 0.12 of the current background parameters value from "Results Log";
  1. Run refine_background.refine background parameters value. The final Rwp should be a little better than before (Rwp = 0.073).

Some improvement on Monte Carlo simulation

It is very hard to verify this part, because all the effort here MAY be able to improve the performance existing MC algorithms that worked already.

So here is what the verification can do.

  1. Download the following files from this ticketâ€™s attachment.
    • NOM_11848-4.dat
    • LB4853b2.hkl
    • nom60b4.irf
    • setup_repview-2.py
    • set_param_limit.py
    • refine_profile.py
  1. Run setup_preview-2.py to a) set up the inputs of LeBailFit() b) observe how the predicted diffraction patter differs from the experimental data by plotting workspace "CalculatedPattern0". You can also find that the Rwp of the initial values is about 0.11 from Results Log.
  1. Run refine_profile.py. Then open TableWorkspace Bank4ProfileTable4_MC2504 to see that parameter Zerot is refined to 140.616.
  1. Run set_param_limit.py to limit Zerot within 100. Open table workspace "Bank4InstrumentParameterTable1_Step2" to check whether 'Max' value of Zerot is 100 now.
  1. Run refine_profile.py again. Read table Bank4ProfileTable4_MC2504 to check that Zerot should be a value within defined range.
  1. Be noticed that the purpose of this test is not to show that limit Zerot within 100 can yield a better Rwp, but to show that LeBailFit is enabled to refine parameter values in a bounded manner in Monte Carlo minimizer.
