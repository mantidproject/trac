Add support for TFXA and TOSCA-1

Reported by: Samuel Jackson Owned by: Dan Nixon
Priority: major Milestone: Release 3.3
Component: Indirect Inelastic Keywords:
Cc: Blocked By: #9960
Blocking: #10512 Tester: Harry Jeffery

Description (last modified by Samuel Jackson) (diff)

These are both older incarnations of TOSCA. We need to create appropriate IDFs and IPFs from the supplied calibration parameters and in consultation with the TOSCA instrument scientists.

As these are older versions of TOSCA, we can just define earlier versions of the TOSCA IDF with the correct date. In Stewart's words:

  • For TFXA only runs >1000 are analysable
  • TOSCA-1 was used for runs TSC00001 to TSC02134

People to talk to about this are Anders and Stewart Parker.

comment:32 Changed 6 years ago by Dan Nixon

To test:

  • See that the TOSCA system test is not broken.
  • Reduce each of the following using the default values on the Indirect > Data Reduction > Energy Transfer UI:
    • TOSCA (e.g. run 9000)
    • TOSCA "DAE" (e.g. run 3000)
    • TOSCA-1 (e.g. run 2000)
    • TFXA (e.g. run 2000)
    • TFXA /w diff detectors (e.g. run 4000)

For all except TOSCA and TOSCA DAE the output should be a single spectra with a single peak around 0 meV, zooming in to just after the peak will show some features from the sample, the plot should usually reach at least 1000 meV.

(optional) Repeat outputting in wave number.

Note this ticket is for Energy Transfer only, diffraction for TFXA is not supported.
