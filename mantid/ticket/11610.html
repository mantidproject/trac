Add documentation (RST) for algorithm SaveNXTomo

Reported by: Federico M Pouzols Owned by: Federico M Pouzols
Priority: major Milestone: Release 3.4
Component: Documentation Keywords:
Cc: Blocked By:
Blocking: Tester: Michael Wedel

Description (last modified by Federico M Pouzols) (diff)

This algorithm is going to change (several entries being written in the output files need to be revisited, and some others to be added). But for now there should be at least basic documentation about what it does.

While this algorithm saves NXTomo files the usual Mantid way (from workspaces), for practical (performance) reasons there's going to be a sibling algorithm to make NXTomo files directly from collection of image files (see related ticket #11041). So better to have all this explained sooner than later.
