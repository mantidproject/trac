SANS: Allow front and rear data to merged STAGE 1

Reported by: Nick Draper Owned by: Anders Markvardsen
Priority: critical Milestone: Release 2.2
Component: Mantid Keywords:
Cc: Richard.heenan@…, stephen.king@… Blocked By:
Blocking: Tester: Martyn Gigg

Description (last modified by Anders Markvardsen) (diff)

From meeting last week Richard/Steve agreed the following user commands should be used to control when data from front and rear detectors:

SYNTAX:

DET/RESCALE r (DEFAULT TO 1.0)

DET/SHIFT s (DEFAULT TO 0.0)

DET/RESCALE/FIT [fromQ toQ]

DET/SHIFT/FIT [Q1 Q2]

Qualifiers:

/RESCALE Scale multiplied to the front data

/SHIFT Constant background subtracted from front data

/RESCALE/FIT Fit RESCALE so front and back data match

/SHIFT/FIT Fit SHIFT so front and back data match

Parameters:

r Value of RESCALE parameter

s Value of SHIFT parameter

Q1 Q2 To optionally specify the Q range to do the fit over. Note if not specified will use the full overlapping front and rear data Q range. If both /RESCALE/FIT and /SHIFT/FIT specify a Q range the latter specified in the user file will be used

In this ticket add code to allow the following additional options in the Analysis Details tab -> Detector bank combobox:

merged rear, front rear, front, merged

The merging and front reduced data are affected by the Front detector Rescale and Shift user input values. In a later tickets these will be allowed to be fitted
