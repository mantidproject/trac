Coverity High Impact issues: resource leaks in MDEvents

Reported by: Federico M Pouzols Owned by: Federico M Pouzols
Priority: major Milestone: Release 3.4
Component: Framework Keywords:
Cc: Blocked By:
Blocking: Tester: Dan Nixon

Description

Well not so incredible high impact, as these are a few memory leaks.

Fix a bunch of resource leak issues found by coverity in Framework/MDEvents: 1075577-1075588 (functions AffineMatrixParameter and createTransform). All of them seem solvable by using boost::shared_ptr and/or deleting pointers to temporary objects.

Change History

comment:1 Changed 6 years ago by Federico M Pouzols

More details about these issues:

  • 1075577-80 are memory leaks in CoordTransformDistanceParser::createTransform that should go away by using shared_ptr.
  • 1075581-83 are similar memory leaks in CoordTransformAffineParser::createTransform
  • 1075584: CoordTransformAffine constructor: leak (pointer going out of scope). This issue is now listed as fixed.
  • 1075585,6: ConvToMDHistoWS: leak (pointer to thread scheduler, be careful there). These two are now listed as fixed, and the file as "historical". Trying to clarify what happened.
  • 1075587: AffineMatrixParameter constructor: memory leak, pointer tmpX going out of scope.
  • 1075588: AffineMatrixParameter copy constructor, similar leak with pointer tmpX.
