Last modified 5 years ago

Inst View: Move grouping functions to Mask tab

Reported by: Nick Draper Owned by: Roman Tolchenov
Priority: critical Milestone: Release 2.5
Component: Mantid Keywords:
Cc: doucetm@…, alex.buts@… Blocked By:
Blocking: Tester: Martyn Gigg

Description (last modified by Martyn Gigg) (diff)

The grouping functionality in the pick tab would be better off in the masking tab. This would give better control of grouping, make more usability sense and consolidate the underlying source code.

Changes:

  1. Rename Mask tab to Mask/Group
  2. Add a Toggle button at the top for "Mask/Group"
  3. In group mode the text for the save as need to change to include/exclude as opposed to ROI/mask
  4. Remove grouping context menu options from pick mode (Group, Mask, Create Grouping File)
  5. Update wiki documentation
  6. As per #6128, the grouping file should not be hardcoded to save to the install bin directory
  7. Check with Mathieu Doucet that these changes are OK as he hosts this tab in one of the sane interfaces.
