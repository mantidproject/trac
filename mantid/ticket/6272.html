Auto Update on SliceViewer

Reported by: Owen Arnold Owned by: Owen Arnold
Priority: major Milestone: Release 2.4
Component: Mantid Keywords:
Cc: Blocked By: #6249
Blocking: #6392 Tester: Stuart Campbell

Description (last modified by Owen Arnold) (diff)

Output from design session in #6249. Would be good to have an auto-update feature similar to that used in ParaView, so that changes in rebinning are automatically applied to a changed view (without having to press the refresh button.)

While we are at it. Another simple change that would be nice, would be to make the 'Apply' button on the SliceViewer look more like the one in ParaView. This might help users go between the SliceViewer and the VSI.

refs #6272 Auto refresh introduced.

An extra button and an extra menu option has been added for this. There are two points that I want to address and check. Firstly, we currently have a lock mode, which is a kind of auto update, but only for when the sliders are moved. Its probably confusing to have two forms of autoupdate, so we should probably roll them into the single lock mode. This should be easy to do. Secondly, if we do not opt for this, it might be best to make the auto refesh icon look something like the current refresh icon, to save confusion.

comment:5 Changed 8 years ago by Owen Arnold

Tester. I'm marking this ticket as done, but I think we'll want to revisit the functionality around auto-updating and re-binning on the SliceViewer at some date to make it simpler. For future reference:

  • The Rebin button rebins only the current slice point within the viewing extents.
  • The refresh button has to be invoked every time the view is panned or zoomed.
  • The auto rebin button automatically applies the refresh.
  • The lock button when unlock auto rebins to the current slice point (as sliders are moved)
