Description (last modified by Owen Arnold) (diff)

1) Needs a "What is this interface for section": Something like:

"This user interface is for managing batch processing of data reduction for reflectometry data. The algorithm is built upon ReflectometryReductionOne. See this algorithm for expansion of the data reduction steps. (reuse workflow diagram here). The GUI works hard to establish instrument level defaults, so that the number of mandatory options are minimal. Conversely, you may provide overrides for anything in the standard algorithm. An optional post-processing step allows you to stitch groups of results together, and plot them, or export them to your favourite data format. Pre-processing and transfer functionality allows data to be imported from various data archives and pre-assembles the batch table."

2) Link to the data did not get generated properly

3) "This is the data before processing." to "This is the data before processing in time of flight." Same for the following usage of workspaces that are referred to as TOF. Infact, please highlght the units for example IvsLam has x units of wavelength in Angstroms-1

4) Options. How do I get to the options.

I don't know if you're familiar with user centred design. Hopefully they are now teaching this at universities. Give one of the other students your documentation, and without telling them how to get it working, or helping them at all, see how they get on. When they get stuck. It's an indication that the documentation could be better. You can do this in iterations, but you need fresh people each time round.
