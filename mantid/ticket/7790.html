Ticket #7790 (closed: fixed)

Last modified 5 years ago

Correct fitting parameter mistake in GEM_Parameters.xml

Reported by: Mike Thomas Owned by: Mike Thomas
Priority: major Milestone: Release 3.0
Component: Diffraction Keywords:
Cc: Blocked By:
Blocking: Tester: Roman Tolchenov

Description

Erroneous code places attribute 'result-unit' in element <parameter> instead of <formula>. Results in incorrect fitting of bank 4 data when using dspacing. The correction has been made and code will be commited.

Wrong code:

<component-link name="bank4" >

<parameter name="IkedaCarpenterPV:SigmaSquared" type="fitting">

<formula eq="176.833*centre2" unit="dSpacing" result-unit="TOF2"/>

</parameter> <parameter name="IkedaCarpenterPV:Gamma" type="fitting" result-unit="TOF">

<formula eq="4.416*centre" unit="dSpacing"/>

</parameter>

</component-link>

Corrected Code:

<component-link name="bank4" >

<parameter name="IkedaCarpenterPV:SigmaSquared" type="fitting">

<formula eq="176.833*centre2" unit="dSpacing" result-unit="TOF2"/>

</parameter> <parameter name="IkedaCarpenterPV:Gamma" type="fitting">

<formula eq="4.416*centre" unit="dSpacing" result-unit="TOF"/>

</parameter>

</component-link>
