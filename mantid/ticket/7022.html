Tester:

You should have a new CMake variable QT_ASSISTANT_FETCH_IMAGES (advanced), this should default to off.

The following testing steps need to be performed at ISIS.

  • Make the documentation with this in it's default off state, and note that the algorithm documentation does not contain any screenshots.
  • Turn it on, and repeat, and again, no screenshots should appear in the documentation.
  • Repeat again, ensuring that the cmake variable is set to on, but this time set the environment variable http_proxy to "​http://wwwcache.rl.ac.uk:8080" prior to the build. Now you should have downloaded dialog images which are linked in the help pages for each of the algorithms.
