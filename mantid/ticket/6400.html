Test environment: Windows 7 64bit
Windows installer 3.2.20141205.1835

No crashes or blocker issues found.

I checked GUI interaction in general and the specific points in functional area group 1. Did Load/Write cycles with different files and formats, checking with CheckWorkspacesMatch.

Issues found:

  • I could reproduce #8307: saving/loading event NeXus data produces an axis mismatch.
    (saving/loading normal NeXus files seems to work well though)
    (if you save again the workspace into a new file, and compare the first loaded workspace with the second one, they match)
  • The tool bar button for the script window is not kept in sync when the script window is closed: #7551
  • Trying to edit the properties of empty 3D plots produces crashes (when deleting the source matrix), see #10713

These issues can also be reproduced in Ubuntu/Debian, they don't seem platform specific.
