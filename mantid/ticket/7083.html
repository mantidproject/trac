PlotAymmetryByLogValue as a function of different parts of log values

Reported by: Anders Markvardsen Owned by: Raquel Alvarez Banos
Priority: minor Milestone: Backlog
Component: Muon Keywords:
Cc: james.lord@… Blocked By: #10894
Blocking: Tester: Anders-Markvardsen

Description (last modified by Anders Markvardsen) (diff)

James writes:

Another feature we could find useful would be to allow choice of average, max/min, first or last (as in ConjoinSpectra). At present I think PlotAsymmetryByLogValue just uses the last value in a Time Series log to get the X numbers. This is OK if we used a script to step the value, allow it to stabilise, then take data. For some future applications we may set the magnetic field / temperature / etc smoothly ramping up or down and then just take a series of short runs during the scan, with no delays for setting and stabilising. In this case the average log is more appropriate. It would have to be the average while RUNNING and ignore the log points taken in any SETUP time prior to the run (these are in the logs too and in other circumstances are very useful to have).
