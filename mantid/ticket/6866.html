Update MantidEV to use new parameter names for OptimizeCrystalPlacement

Reported by: Dennis Mikkelson Owned by: Dennis Mikkelson
Priority: major Milestone: Release 2.5
Component: Mantid Keywords:
Cc: mikkelsonr@…, petersonpf@…, lynchve@… Blocked By:
Blocking: Tester: Martyn Gigg

Description

The names of several parameters to OptimizeCrystalPlacement were changed. As a result, the property names used by MantidEV when setting parameters to call OptimizeCrystalPlacement need to be updated to match, to avoid an exception when MantidEV trys to set those parameters.

  • Status changed from accepted to verify
  • Resolution set to fixed

NOTES TO TESTER:

1.The branch is: feature/6866_FixMantidEVPropertyNames

2.The attached file, TOPAZ_3007_BROKEN.peaks is a modified version of the file TOPAZ_3007.peaks in the AutoTestData directory. The goniometer angles have all be altered slightly.

To Test:

--Bring up the "SCD Event Data Reduction" interface (aka. MantidEV).

--Switch to the second tab, Find Peaks, enter a name for the peaks workspace, select "Load ISAW Peaks(or Integrate) File, and select the attached TOPAZ_3007_BROKEN.peaks file. Press Apply.

--Switch to the third tab, Find UB, and select "Load Isaw UB" and select the file TOPAZ_3007.mat from the AutoTestData directory. Also, turn on the "Optimize Phi, Chi and Omega", and the "Index Peaks Using UB" options. Press Apply.

The results log should show that only 5 peaks were originally indexed by the matrix, with the bad goniometer angles, but after the optimization and indexing, all 43 peaks were indexed.
