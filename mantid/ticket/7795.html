Last modified 5 years ago

Correct coordinate attribute error in multiple IDFs

Reported by: Mike Thomas Owned by: Mike Thomas
Priority: major Milestone: Release 3.0
Component: Framework Keywords:
Cc: Blocked By:
Blocking: Tester: Andrei Savici

Description

The error uses the attribute 'axis' instead of 'value' for the element <handedness>. Some IDFs that are written by other software have had this error replicated. This is being dealt with separately, see: ​https://github.com/mantidproject/mantidgeometry/issues/12

Erroneous Code:

<reference-frame>

<along-beam axis="z"/> <pointing-up axis="y"/> <handedness axis="right"/>

</reference-frame>

Replacement Code:

<reference-frame>

<along-beam axis="z"/> <pointing-up axis="y"/> <handedness val="right"/>

</reference-frame>
