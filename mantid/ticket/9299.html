Visible Time Slicing

Reported by: Nick Draper Owned by: Nick Draper
Priority: critical Milestone: Release 3.4
Component: GUI Keywords: SSC,2014,All
Cc: Blocked By:
Blocking: Tester: Nick Draper

Description

Should create a reusable widget (stand-alone and docked in other guis). Allow for (over)plotting log values, selecting ranges to keep/delete, and looping over log values (e.g. temperature every 5 degrees) and annotating the filter on the plot. Needs lots of planning/design. The plotting is the big unknown.

"Also from SANS A tool that allows us to time slice a file into a certain number of linear or logarithmic steps without us having to know/look up the length of the run. At present we have to find the file (in the journal file normally), work out how long it was, calculate the step size we want and then use the slice tool. It would be handy (for a quick check to see if something is changing with time) for Mantid to slice a file in half or into 8 (or whatever number you want, lin or log) without having to check the length of a run.

and

A generic filter tool which would allow us to filter out or only use data between specific values e.g. between certain temperatures, when the fast shutter is open, remove the data when the fast shutter was closed etc. If we log the appropriate values then they should be in the nxs file for us to filter on.

"
