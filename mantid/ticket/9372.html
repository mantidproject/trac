"Previously-Entered Value" Icon Does Not Dissapear When Value is Changed Using "Browse"

Reported by: Peter Parker Owned by: Peter Parker
Priority: trivial Milestone: Release 3.2
Component: GUI Keywords:
Cc: Blocked By: #9437
Blocking: Tester: Jay Rainey

Description

For FileProperties with a "Browse" button, the "previously-entered value" icon introduced in #2480 does not disappear when the "Browse" button is used to change to a different file.

Moving the focus to a different property field, then back to the FileProperty field, then back out again seems to give the icon the "nudge" it needs and it disappears.

A small bug, but could do with fixing.
