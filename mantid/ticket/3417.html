comment:6 Changed 9 years ago by Russell Taylor

I strongly disagree!

OK, it would be better if the directory was not called ISIS, but it is a hidden directory! We really shouldn't be bothering users with this kind of thing, even if we could assume they will read the release notes. And also, on Windows we'd be asking them to edit the registry.

A better solution would be to have this handled programatically, either in the MantidPlot code or the installers. Even then, the should be balanced against the time (=money) it would take.
