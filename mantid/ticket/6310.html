Update POLARIS definitions

Reported by: Martyn Gigg Owned by: Anders Markvardsen
Priority: blocker Milestone: Release 2.4
Component: Mantid Keywords:
Cc: Blocked By:
Blocking: Tester: Nick Draper

Description

Ron Smith has some changes for the POLARIS definitions that are currently only on the instrument computer. We need to check & ship them with Mantid.

Message from Ron:

I've had a look at the Polaris file which installed with the latest version of Mantid.It is correct in as much as it has all the detector 2theta angles and UDETs defined correctly.

However, its "valid-from" date is incorrect because during cycles 2011/4 and 2011/5, during which time we collected a number of instrument commissioning data sets, the 2theta against UDET mapping was the reverse of what it ought to have been due to the misunderstanding I mentioned to you between the design engineers, Julian Norris' people and me.

So attached are 2 files:

  1. The first ("POLARIS_Definition08-03-2012#11-43_121.xml") is the current working version of the Polaris Instrument description File before our reinstall today, which was on the PC in the Polaris cabin. It has "_121" appended to my date and time-stamped filename, denoting to me that it's valid for ISIS cycles 2012/1 (which started on 14th May) onwards. In order to allow it to work for any RAW data files that may have been created in the days leading up to the start of that cycle I edited it by hand to make it valid from 1st May (i.e. 2012-05-01T11:43:38). Note that I kept the hh:mm:ss time unchanged so that I could still identify on what date (8th March 2012) I actually created the file....
  1. The second file ("POLARIS_Definition_115.xml") is a modified version of the first one, with the Bank 1 idlist defining the UDETs numbered in the reverse order. It has "_115" appended to its filename to indicate to me that it is valid for that cycle. In the file itself, it is flagged as valid from 1st January 2011 to 30th April 2012.

Please add these two files to the Mantid distribution in place of the existing one. They are missing 3 or 4 XML header lines, which appear on the files that are distributed with Mantid. I did try to put these first few lines in with my Fortran code which created the files, but I must have had a mistake in it as my they caused a crash.
