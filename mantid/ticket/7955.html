Muon: Dead Time file logic is broken

Reported by: Arturs Bekasovs Owned by: Arturs Bekasovs
Priority: major Milestone: Release 3.0
Component: Muon Keywords:
Cc: Blocked By:
Blocking: #6691 Tester: Gesner Passos

Description

To reproduce:

  1. Initially, set Dead Time Table to "None".
  2. Load and plot attached EMU00036854.nxs.
  3. Now set Dead Time Table to "From Disk", write some nonsense inside the box, e.g. "non-existent file", and press Enter. Message box saying "A problem occurred while applying dead times." should appear, which is expected.
  4. Now, click "Browse" near the Dead Time file box, and select attached dtepar.nxs. The same message box appears again. This shouldn't happen, because this IS a valid Dead Time file.
  5. Try to select the same file again. This time correction is applied without any errors.
  6. Write some non-sense and press Enter again - the correction is applied without errors, though the file obviously does not exist.
